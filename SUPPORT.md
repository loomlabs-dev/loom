# Loom Support

Use the right channel for the right problem.

## Read First

Before asking for help, check:

- [README.md](README.md)
- [ROADMAP.md](ROADMAP.md)
- [CONTRIBUTING.md](CONTRIBUTING.md)
- [TRADEMARKS.md](TRADEMARKS.md)
- [SECURITY.md](SECURITY.md)
- [docs/alpha/README.md](docs/alpha/README.md) for the current alpha posture

## Where To Go

### 1. Bug reports

Use GitHub Issues for normal product or docs bugs on the active mainline.

### 2. Feature requests

Use GitHub Issues for product proposals that strengthen Loom's current
direction.

### 3. Security issues

Do not open a public issue. Follow [SECURITY.md](SECURITY.md).

### 4. Historical questions

If you truly need older product context, inspect git history instead of
treating pre-reset material as active guidance.

## What To Include In Any Help Request

- Loom surface affected
- Loom branch, tag, or commit
- exact commands run
- the error or unexpected behavior
- any screenshots or public-site context when relevant
